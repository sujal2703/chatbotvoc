# AI Assistant — Prompt Engineering Project

A small Flask web app that acts as a multi-function AI assistant: it answers
questions, summarizes text, generates creative content, and gives advice —
each with three different underlying prompt styles you can compare, plus a
simple feedback loop to track which styles actually perform well.

## Setup

1. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

2. Add your OpenAI API key:
   - Make a copy of `.env.example` and rename it to `.env` (same folder as `app.py`).
   - Open `.env` in any text editor and replace `your_key_here` with your real key, so the line reads:
     ```
     OPENAI_API_KEY=sk-...your-actual-key...
     ```
   - Save the file. The app loads it automatically on startup — no terminal commands needed.
   - (Alternative: you can still set it as a real environment variable instead — `export OPENAI_API_KEY=your_key_here` on Mac/Linux, or `$env:OPENAI_API_KEY="your_key_here"` on Windows PowerShell — but the `.env` file is the easier route.)

3. Run the app:
   ```bash
   python app.py
   ```

4. Open **http://127.0.0.1:5000** in your browser.

## How the project requirements map to the code

| Requirement | Where it lives |
|---|---|
| **Functionality** (3+ distinct functions) | `prompts.py` → `FUNCTIONS` dict defines Answer Questions, Summarize Text, Generate Creative Content, and an optional 4th (Give Advice). |
| **Prompt Design** (3 prompt variants per function, varying length/tone/complexity) | `prompts.py` → each function has a `direct/detailed/casual`-style set of three named prompt templates, with a comment explaining what makes each one different. |
| **User Interaction** (pick a function, submit input, see output) | `app.py` → `index()` route, rendered by `templates/index.html`. The interface lets you pick the function, pick the prompt style, type input, and see the response inline. |
| **Feedback Loop** (yes/no on each response, used to refine prompts) | `app.py` → `submit_feedback()` writes each rating to `feedback_log.json`; `feedback_summary()` + `templates/feedback_summary.html` show helpful-rate stats broken down by function and by prompt variant, so you can see which variants are actually landing. |
| **Documentation** | This README explains setup + mapping. **The PPT user guide is a separate deliverable** — build that separately summarizing the app's functions and how to use them; it isn't generated by this code. |

## Project structure

```
ai-assistant/
├── app.py                   # Flask routes
├── prompts.py                # All prompt templates, 3 variants per function
├── llm_client.py              # OpenAI API wrapper + error handling
├── feedback_log.json          # Feedback entries accumulate here at runtime
├── templates/
│   ├── index.html             # Main interface
│   └── feedback_summary.html  # Feedback dashboard
├── static/
│   └── style.css
├── requirements.txt
└── .env.example
```

## Notes

- Session history is stored in memory only (resets when the server restarts) — that's intentional, this is a single-demo-session project, not a production app with accounts.
- If the API key is missing or invalid, or the API call fails, the app shows a friendly error message instead of crashing.
- To reset feedback stats, just empty `feedback_log.json` back to `[]`.
